Point Cloud Segmentation and Feature Extraction

Features

  • Given Point Cloud, Segmentation into various classes such as buildings, trees etc.
  • Generating a roof top view as shape file for building
  • Extracting the polygons for each building and roof-top feature (In terms of planar polygons)
  • Extracting trees location,height,Area and 2D polygon shape of top view
  • For better visualization , creating seperate point clouds for trees and building planes.
  • Extracted Data is stored in Shapefiles and JSON files.

Results after running all the code files

  • Step1 saves a model file which consist of model parameters for point cloud segmentation with .sav extension , a pretrained model is provided in the directory. If training from scratch , uncomment train function in code.
  • Step2 returns a feature file .npy and classfied LAS file with name as classified_+lasfile_name.las
  • Step3 returns following things :
    • Normalized LAS file with z coordinate replaced with actual height of any point grounded_file.las.
    • A Tree Only LAS file according to the classification in previous step full_trees.las.
    • To curb with some misclassification for tree , filtered LAS file trees_main_fil_*.las is created with color thresholds.
    • trees_colored.las , LAS file with each tree cluster having different color coding.
    • tree_parameters.npy , contains parameters (height,location,area,polygon coordinates) for each tree cluster.
    • polygons_60.shp/dbf/shx , shapefile for polygons as ConvexHull of trees.
    • tree_data.json , contains parameters in JSON format.
  • Step4 returns following things :
    • A Building Only LAS file according to the classification in previous step Buildings.las.
    • all_500.npy file which contains normals calculated for each cube.
    • planes_500.npy file which contains 3D planes calculated with ConvexHull method from scipy.
    • planes_2D_500.shp/dbf/shx file which are the shapefiles for rooftop view of buildings.
    • final_planes_3D_\heights_500.npy contains polygon parameters.
    • polygon_data.json which contains 3D polygon coordinates and height (lowest,highest) for each plane.
  • Step5 returns following things :
    • normal_cur_main.npy which contains normal and curvature for each point in Building.las Pointcloud.
    • regions_main.npy which contains major and minor regions after clustering step.
    • final_regions_planes_*_*_pca.npy which contains regions after combining the previous major and minor regions.
    • plane_shapes.shp , shapefile for planes extracted for each building feature.
    • plane_*_*_pca.las which is the LAS file with each plane surface is represented with different color coding for better visulization.
    • planes_data.json , contains height(upper,lower) for each plane with the 3D coordinates.
  • Step6 returns a object file planes_CAD.off which is a water tight model generated for building with previous step as exact building planes were found.
